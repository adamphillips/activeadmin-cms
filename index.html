Active Admin CMS

ActiveAdmin CMS is an easy way to add simple CMS functionality to ActiveAdmin for Ruby on Rails applications.

What does it do

ActiveAdmin CMS allows you to define recipes for CMS content using a simple Ruby DSL

For example, a simple recipe might look like:

module Recipes
  class Default < Cms::Recipe
    
    define do
      section :main_content do
        image :title_image, :hint => 'An image containing the title of the page'
        string :sub_heading, :hint => 'The text that appears in bold under the title'
        text :content, :hint => 'The main copy for the page'
      end

      section :side_bar do
        image :image_1
        image :image_2
      end
    end

  end
end

This defines the administration screen in ActiveAdmin for pages using this recipe.

ActiveAdmin edit screen for the above recipe

A recipe is made up of a number of different pieces of content, which can be one of a number of types including markdown formatted text and images. You can also define additional custom types if you need them.

To render the html for the entered content is as simple as using

cms_content('main_content:title_image')

in your view

What doesn't it do?

ActiveAdmin CMS aims to cater for small but design-intensive websites. Therefore it leaves developer-centric tasks, such as the defining of routes, to the developer.

If you are looking for a CMS to manage your 1000 page website where you can just apply a one-size-fits-all template to the entire website, then this probably isn't for you. On the other hand, if you have a 10 page website but each page is specifically crafted with different CMS requirements this could be exactly what you need
